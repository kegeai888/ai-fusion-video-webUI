## 1. 启动平台

在服务器上执行：

```bash
# 确保 MySQL 和 Redis 已运行
sudo service mysql start
sudo service redis-server start

# 启动前后端
cd /root/ai-fusion-video
./start_app.sh
```

启动完成后，在浏览器访问：

```
http://<服务器IP>:7860
```

> 日志位置：
> - 后端：`ai-fusion-video/backend.log`
> - 前端：`ai-fusion-video-web/frontend.log`

## 2. 首次初始化

**首次访问**时，系统会自动跳转到初始化页面 `/setup`。

填写以下信息创建管理员账号：

| 字段 | 说明 |
|------|------|
| 管理员用户名 | 登录用，建议英文 |
| 昵称 | 显示名称 |
| 密码 | 至少 6 位 |
| 确认密码 | 与密码一致 |

点击 **完成初始化**，系统自动跳转登录页。

> 初始化只需执行一次，之后直接登录即可。

## 6. 项目管理

### 新建项目

1. 点击顶部 **项目** 或仪表盘 **新建项目**
2. 填写项目信息：

| 字段 | 说明 |
|------|------|
| 项目名称 | 必填 |
| 描述 | 选填 |
| 项目类型 | 漫剧 / 短剧 / 动画 / 纪录片 / 宣传片 / MV |
| 画面比例 | 横屏 16:9 / 竖屏 9:16 等 |
| 画风 | 选择艺术风格预设 |

3. 点击 **创建**

### 项目列表操作

- **搜索**：按名称筛选项目
- **进入项目**：点击项目卡片
- **删除项目**：点击项目卡片右上角菜单 → 删除

### 项目概览

进入项目后，**概览页** 显示：
- 项目基本信息与设置
- 创作进度：剧本 → 分镜 → 资产 三段式流程状态

## 7. 剧本创作

### 进入剧本页

项目侧边栏 → **剧本**

### 创建剧本

**方式一：手动创建**
1. 点击 **新建剧本**
2. 输入剧本标题，点击确认
3. 在编辑器中手动添加分集和场次

**方式二：AI 解析剧本**
1. 点击 **AI 解析剧本**
2. 输入剧本标题和剧本原文（纯文本）
3. 点击 **解析**，AI 自动拆分分集和场次

### 剧本结构

```
剧本
└── 分集（Episode）
    └── 场次（Scene）
        └── 场次内容（台词、动作描述等）
```

### 剧本编辑操作

| 操作 | 说明 |
|------|------|
| 添加分集 | 点击分集列表底部 **+** |
| 添加场次 | 选中分集后点击 **添加场次** |
| 删除分集/场次 | 右键菜单或操作按钮 |
| 拖拽排序 | 拖动分集/场次调整顺序 |
| AI 解析单集 | 选中分集 → **AI 解析** |

## 8. 分镜制作

### 进入分镜页

项目侧边栏 → **分镜**

### 分镜结构

```
分镜
└── 分集
    └── 场次
        └── 镜头条目（Storyboard Item）
            ├── 镜头描述
            ├── 参考图
            └── 生成图/视频
```

### 视图切换

- **表格视图**：适合批量查看和编辑
- **卡片视图**：适合查看图像效果

### 分镜操作

| 操作 | 说明 |
|------|------|
| 新建镜头 | 点击场次下 **添加镜头** |
| 批量创建 | 选中多个场次 → **批量生成镜头** |
| 调整顺序 | 拖拽镜头条目排序 |
| AI 生成图片 | 选中镜头 → **生成图片**（需配置 AI 模型） |
| AI 生成视频 | 选中镜头 → **生成视频**（需配置视频生成模型） |
| AI 工具刷新 | 顶部 **AI 工具** 按钮，批量重新生成 |

## 10. AI 生成功能

> 使用 AI 功能前，需先在 **系统设置 → AI 模型** 中配置 API Key。

### AI 剧本解析

在剧本页，将已有剧本文本粘贴后，AI 自动识别分集和场次结构。

### AI 图片生成

在分镜页选中镜头条目，点击 **生成图片**：
1. 系统根据镜头描述和项目画风自动构建提示词
2. 调用配置的图片生成模型（如 DashScope、OpenAI DALL-E 等）
3. 生成完成后图片显示在镜头卡片上

### AI 视频生成

在分镜页选中已有参考图的镜头，点击 **生成视频**：
1. 基于参考图生成短视频片段
2. 任务异步执行，可在 **任务通知** 面板查看进度

### AI 助手

点击右上角 **AI 助手** 图标，打开对话面板：
- 支持多轮对话
- 可询问创作建议、剧情优化等
- 历史会话保存，可随时查看

### 任务通知面板

点击右上角 **通知/任务** 图标：
- 查看所有 AI 生成任务的状态（运行中 / 完成 / 失败）
- 取消正在运行的任务

## 11. 系统设置

### 通用设置

**系统设置 → 通用**：配置平台访问域名（用于生成媒体资源的完整 URL）。

### 个人设置

**系统设置 → 个人**：修改昵称、邮箱、手机号、密码。

### AI 模型配置

**系统设置 → AI 模型**：

1. 先在 **API 配置** 中添加 API Key：
   - 选择供应商（OpenAI / Anthropic / DashScope / Ollama 等）
   - 填写 API Key 和 Base URL
   - 点击 **测试连接** 验证

2. 在 **模型配置** 中为不同功能指定使用的模型：
   - 文本生成模型（剧本解析、AI 助手）
   - 图片生成模型（分镜图片生成）
   - 视频生成模型（分镜视频生成）

### 存储配置

**系统设置 → 存储**：

| 存储类型 | 说明 |
|---------|------|
| 本地存储 | 默认，文件保存在服务器 `./data/media` 目录 |
| S3 兼容存储 | 支持 AWS S3、阿里云 OSS、腾讯云 COS、MinIO 等 |

配置 S3 存储需填写：Endpoint、Bucket、Access Key、Secret Key、Region。

## 12. 常见问题 FAQ

**Q：登录时提示 Network Error？**  
A：检查服务是否正常启动。在服务器执行 `./start_app.sh`，确认前后端均已启动。若问题持续，检查 MySQL 和 Redis 是否运行：`sudo service mysql start && sudo service redis-server start`。

**Q：AI 生成功能无响应或报错？**  
A：进入 **系统设置 → AI 模型 → API 配置**，确认已添加有效的 API Key，并点击 **测试连接** 验证。

**Q：上传图片失败？**  
A：检查 **系统设置 → 存储** 中的存储配置是否正确。本地存储模式下，确认服务器 `./data/media` 目录有写入权限。

**Q：如何查看后端报错日志？**  
A：查看 `ai-fusion-video/backend.log`，或执行：
```bash
tail -f /root/ai-fusion-video/ai-fusion-video/backend.log
```

**Q：前端页面空白或加载异常？**  
A：查看前端日志：
```bash
tail -f /root/ai-fusion-video/ai-fusion-video-web/frontend.log
```
也可尝试重启服务：`./start_app.sh`。

**Q：忘记管理员密码怎么办？**  
A：请联系系统管理员，通过后台用户管理功能或专用维护脚本重置密码。不要直接复制网上的 SQL 修改密码字段，因为密码使用 BCrypt 加密，写入错误哈希会导致账号无法登录。

**Q：如何添加新用户？**  
A：目前支持通过注册接口创建账号（`/api/auth/register`）。管理员用户管理功能在 **系统设置 → 用户管理** 中（需管理员权限）。

**Q：分镜图片生成很慢？**  
A：图片生成依赖外部 AI 服务，速度取决于所配置的 API 供应商响应时间。可在 **任务通知** 面板查看任务进度，无需等待页面响应。

**Q：项目数据保存在哪里？**  
A：所有项目数据保存在 MySQL 数据库中，媒体文件保存在配置的存储位置（本地或 S3）。建议定期备份数据库。

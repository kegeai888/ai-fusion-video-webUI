# 用户使用手册

## 1. 项目简介

本项目为“融光 AI 视频创作平台”，当前提供：

- 前端 Web 页面
- 后端 API 服务
- MySQL 数据库依赖
- Redis 缓存依赖

当前已经适配特殊容器环境，可通过 shell 脚本完成一键启动，并可放入 `/start.d` 实现开机自动运行。

---

## 2. 目录中的关键脚本

### `boot-start.sh`

完整启动入口脚本，负责：

- 检查/启动 MySQL
- 检查/启动 Redis
- 注入后端运行环境变量
- 调用 `start_app.sh` 启动前后端
- 启动后做前后端健康检查

这是当前**推荐使用的一键启动脚本**。

### `start_app.sh`

应用层启动脚本，负责：

- 清理前端 7860 端口占用
- 清理后端 18080 端口占用
- 启动 Spring Boot 后端
- 启动 Next.js 前端
- 输出启动状态与日志位置

适用于中间件已经准备好的场景。

### `test_login.sh`

联调验证脚本，负责：

- 检查前端是否可访问
- 检查后端是否可访问
- 检查未登录时认证拦截是否正常
- 验证登录接口
- 验证 accessToken 是否有效
- 验证用户信息接口是否正常

---

## 3. 启动方式

## 3.1 推荐：一键启动整套系统

执行：

```bash
bash /root/ai-fusion-video/boot-start.sh
```

该命令会尽量自动完成：

1. MySQL 可用性检查
2. Redis 可用性检查
3. 本机服务复用
4. 本地 service 启动
5. Docker Compose 中间件兜底
6. 前后端启动
7. 启动结果校验

适用场景：

- 日常启动整套服务
- 开机自启脚本调用
- `/start.d` 自动执行
- 新环境初始化后的首次联调

---

## 3.2 仅启动前后端应用

如果 MySQL 和 Redis 已经在运行，可直接执行：

```bash
bash /root/ai-fusion-video/start_app.sh
```

适用场景：

- 中间件已经就绪
- 只需要重启前后端
- 调试前端/后端启动问题

---

## 3.3 开机自动启动方式

当前环境不依赖 `systemd`，建议做法是：

1. 将启动脚本放入 `/start.d`
2. 赋予脚本可执行权限
3. 由系统在开机时自动执行

示例：

```bash
cp /root/ai-fusion-video/boot-start.sh /start.d/boot-start.sh
chmod +x /start.d/boot-start.sh
```

说明：

- 若复制到 `/start.d`，请确保脚本运行目录中仍能找到：
  - `start_app.sh`
  - `ai-fusion-video/`
  - `ai-fusion-video-web/`
- 更稳妥的方式是让 `/start.d` 中的脚本调用项目中的原始脚本

例如：

```bash
#!/usr/bin/env bash
bash /root/ai-fusion-video/boot-start.sh
```

```text
http://127.0.0.1:18080/api/v3/api-docs
```


如果二者表现不一致，优先以当前实际运行的 context path 为准。

## 6. 启动验证方法

推荐启动完成后执行：

```bash
bash /root/ai-fusion-video/test_login.sh
```

该脚本会依次验证：

1. 前端首页可达
2. 后端接口可达
3. 未登录时受保护接口会被拦截
4. 登录可成功获取 accessToken
5. Token 可正常访问用户信息接口

如果需要自定义账号密码：

```bash
bash /root/ai-fusion-video/test_login.sh 用户名 密码
```

例如：

```bash
bash /root/ai-fusion-video/test_login.sh admin admin123
```

## 8. 常见问题排查

## 8.1 前端启动失败，提示 7860 端口被占用

常见现象：

```text
EADDRINUSE: address already in use 0.0.0.0:7860
```

原因：

- 上一次 Next.js 开发进程未正常退出
- 旧的前端监听还残留在 7860

处理建议：

1. 先重新执行：

```bash
bash /root/ai-fusion-video/start_app.sh
```

当前脚本已内置端口清理逻辑。

2. 如果仍失败，检查 `frontend.log`

3. 必要时手动排查监听进程：

```bash
lsof -i:7860
```

---

## 8.2 后端启动失败

优先检查：

- MySQL 是否可连接
- Redis 是否可连接
- `backend.log` 是否出现配置绑定失败
- 18080 端口是否已被旧 Java 进程占用

建议查看：

```bash
lsof -i:18080
cat /root/ai-fusion-video/backend.log
```

---

## 8.3 启动后前端能打开，但接口不可用

优先排查：

- 后端是否真的启动成功
- 前端代理是否已正确转发到后端
- 登录接口路径是否正常
- 后端 context path 是否与代理目标一致

建议先执行：

```bash
bash /root/ai-fusion-video/test_login.sh
```

---

## 8.4 MySQL / Redis 没启动怎么办

`boot-start.sh` 已经会尝试：

1. 使用现有实例
2. 启动本机 service
3. 必要时用 Docker Compose 拉起中间件

因此建议优先用：

```bash
bash /root/ai-fusion-video/boot-start.sh
```

而不是直接手工启动前后端。
